# PROJETO MESTRE
### A1 Game Academy

## Missão
Ensinar Python através do desenvolvimento de jogos completos, do zero ao fim,
documentando todo o processo de forma estruturada.

## Objetivos Triplos
Todo projeto da Academia serve três propósitos simultâneos:

1. **Aprender** Python na prática, através de um projeto real.
2. **Documentar** esse aprendizado de forma estruturada, gerando material para vídeos.
3. **Transformar** esse material, ao longo do tempo, em cursos e ebooks.

Nenhuma decisão técnica deve considerar apenas "fazer funcionar" —
deve considerar também se ensina bem.

## O que define um Projeto da Academia
Todo projeto que nasce sob a A1 Game Academy segue:

- desenvolvimento por **Sprints**, documentadas conforme `METODOLOGIA.md`;
- **repositório próprio** no GitHub (separado do repositório institucional),
  compondo o portfólio;
- referência obrigatória à metodologia oficial da Academia — nenhum projeto
  cria seu próprio processo isolado.

## Estrutura de Repositórios
A documentação institucional (metodologia, padrões, administração) vive no
repositório `a1-game-academy`. Cada projeto/jogo vive em seu próprio
repositório, referenciado em `docs/projetos/` do repositório institucional.

Ver `ROADMAP.md` para a lista de projetos e fases.
